# Режим «Рейд» (экстракшн, PvEvE) — Фаза 1

Экстракшн-лутер-шутер в духе Escape from Tarkov поверх инфраструктуры арены.
Кор-луп: **заход → лут/бои → экстракт по таймеру = выжил со шмотом; смерть = шмот остаётся
лутом на локации.** Раунд-скоуп: стэш/лут живут только в пределах раунда сервера
(персистентности между раундами нет — так выбрано на этапе дизайна).

## Что уже работает (Фаза 1)

- `RaidController` — предзагрузка карты рейда, таймер рейда, объявления старта/конца.
- `RaidEntryButton` — вход с хаба: переносит мобов с грида кнопки на спавны рейда, стартует таймер.
- `RaidSpawnMarker(1..3)` — точки появления (инсерт).
- `RaidExtractionPoint` — зона экстракта: стой `extractTime` сек → эвакуация на хаб (с попапом %).
- `RaidReturnMarker` — точка возврата выживших на хабе.
- Потеря шмота при смерти — штатное поведение SS14 (тело роняет инвентарь), отдельного кода нет.

Код: `Content.Server/_Wega/Raid/` (компоненты + `RaidControllerSystem` + `RaidExtractionSystem`).
Прототипы: `Resources/Prototypes/_Wega/Entities/Markers/Spawners/Conditional/raid.yml`.
Локаль: `Resources/Locale/ru-RU/_wega/raid/raid.ftl`.

## Что уже работает (Фаза 2 — экономика)

Тарковский цикл «вынес лут → получил деньги → закупился»:

- **Лут на карте** — спавнеры `RaidLootSpawnerCommon/Rare/Epic` (в `raid_loot.yml`): по загрузке карты
  с шансом спавнят случайные предметы, метят их `RaidLoot` и самоудаляются. Стоимость предметов —
  штатная карго-оценка (`PricingSystem`).
- **Награда за вынос** — при экстракте суммируется стоимость вынесенной добычи (с меткой `RaidLoot`),
  лут «продаётся» (списывается) и игрок получает **физические телекристаллы** рядом с собой на хабе.
  Стартовое снаряжение метки не имеет — остаётся при игроке. ТК физические → теряются с трупа при
  смерти (по-тарковски). Настройки в `RaidController`: `creditsPerTc`, `baseReward`, `maxReward`.
- **Хаб-магазин** — `RaidShopTerminal` (на базе `StorePresetFullArsenal`): весь арсенальный пул за ТК.
  Игрок вставляет заработанные ТК и закупается перед следующим заходом.

Код Фазы 2: `RaidLootComponent`, `RaidLootSpawnerComponent`/`System`, расширенный `RaidControllerSystem`
(`ExtractRaider` считает и продаёт лут, выдаёт ТК).

## Как собрать режим (маппинг)

1. **Сделай карту-локацию рейда** в ин-гейм редакторе (как делаются арены) и сохрани, например,
   в `Resources/Maps/_Wega/Raid/raid_location.yml`. На ней расставь:
   - несколько `RaidSpawnMarker`, `RaidSpawnMarker1`, … (точки инсерта по краям);
   - одну или несколько `RaidExtractionPoint` (зоны выхода, обычно по углам/в укрытиях).
2. **На хаб-карте** поставь:
   - один `RaidController` и в его поле `raidMap` пропиши путь к карте из п.1;
   - `RaidEntryButton` (кнопка входа, можно несколько);
   - `RaidReturnMarker` (куда возвращать выживших).
3. Запусти раунд, встань на грид кнопки, нажми её — телепортнёт в рейд, пойдёт таймер.
   Встань на `RaidExtractionPoint` и подожди — эвакуирует обратно на хаб.

> Базовый путь карты в прототипе (`/Maps/_Wega/Raid/raid_location.yml`) — заглушка.
> Пока такой карты нет, контроллер залогирует ошибку загрузки и вход работать не будет.

## Настройки

- `RaidController.raidDuration` — длительность рейда (сек, по умолчанию 900 = 15 мин).
- `RaidExtractionPoint.range` / `extractTime` — радиус зоны и время стояния для эвакуации.

Маппинг: расставь по локации `RaidLootSpawnerCommon/Rare/Epic` (или один `RaidLootField` для
авто-разброса по всему гриду), на хаб — `RaidShopTerminal`.

## Что уже работает (Фаза 3 — AI/скавы, PvEvE)

- **Фракции** (`_Wega/ai_factions.yml`): `Skav` и `RaidBoss`. Скавы враждебны к игрокам
  (`NanoTrasen`/`Syndicate`), но **не к боссам и не к своим** (нейтралитет = отсутствие в `hostile`).
  Боссы тоже бьют игроков, но скавов не трогают.
- **Дикие** (`_Wega/Entities/Mobs/NPCs/skav.yml`): один тип `MobSkav` со **случайным снаряжением** —
  `Loadout` выдаёт ОДИН случайный набор из `SkavGear*` на каждом спавне (мачете / топор / копьё / нож /
  тулбокс / кустарный пистолет / ПП), не фикс-пресет. Медленные. Меткость задаётся **типом**
  (`RaidSkavAccuracy`: косой конус 50°, медленный доворот), а не стволом; стволы — «изношенные»
  (`*Skav`, дешёвые). Поведение (`RaidSkavCompound`): бегство при ранах → бой → **падальщичество**
  (ищут и подбирают валяющиеся предметы) → **патруль** (блуждание). В рюкзаке ценность + запасные
  магазины — падают с трупа.
- **Огнестрел у Диких** (`RaidSkavRangedCompound` + `RaidSkavAutoReload`): стреляют пока есть патроны;
  магазин кончился → сами достают запасной из сумки и перезаряжаются; запасные кончились → ствол
  пустой → ближний бой прикладом/кулаками. Без ванильного цикла «бросил-поднял пустой ствол»
  (кастомный ранед-компаунд без веток дропа/подбора). Босс — то же, но точный.
- **Боссы** (`MobSkavBoss`/`MobSkavBossShotgun`/`MobSkavBossSMG`): толстые (500 HP), точные (обычные
  стволы — АК/дробовик/Дрозд), **стоят и охраняют точку** (`RaidBossCompound` без блуждания), имя
  случайно-уникальное («Вожак «Бешеный Кабан»» через `RandomMetadata` + датасеты), снаряжение всегда
  лучше (бронежилет, шлем). Бонус-дроп с трупа: золото, алмаз, АК.
- **Болтовня** (`RaidSkavChatterComponent`/`System`): живые скавы изредка выкрикивают случайные
  реплики («Чую мясо...», «Лут! Моё!»), боссы — свои, реже и весомее. Интервал/шанс рандомные.
- **Боевые барки + зов стаи** (`RaidSkavCombatReactionSystem`): в бою скав выкрикивает боевую
  реплику («Враг! Бей его!») и «зовёт» соседних скавов в радиусе 12 — добавляет им цель в память
  агра (`NPCRetaliation`+`FactionException`), чтобы стая запоминала и преследовала жертву (в т.ч.
  после потери из вида). Кулдаун против спама.
- **Бегство** (`RaidSkavCompound` + `LowHealthPrecondition`): дикий скав, набрав ~половину урона до
  крита, отходит от врага на 10–14 тайлов (`MaintainRangeOperator`) вместо драки. Босс НЕ бежит
  (его `RaidBossCompound` без ветки бегства) — стоит насмерть.
- **Лут с трупа** (`RaidLootCarrierComponent`/`System`): при смерти скава всё его снаряжение метится
  `RaidLoot` — игрок может снять с трупа и вынести за деньги. Босс дополнительно роняет
  гарантированный бонус (золото/алмаз). Death-акидайзер скавам намеренно не ставится.
- **Спавнеры** (`raid_skav.yml`): `RaidSkavSpawner` (с шансом — стрелок), `RaidSkavBossSpawner` —
  на базе штатного `RandomSpawner`, спавнят при загрузке карты и самоудаляются.

## Что уже работает (Фаза 1.1 — напряжение и корректность)

- **Предупреждения о таймере**: за `warningTimes` секунд до конца (по умолчанию 300 и 60) идёт
  объявление «на выход». Настраивается на `RaidController`.
- **Смерть по таймеру (MIA)**: кто не успел эвакуироваться к концу рейда — добивается `miaDamage`
  (по умолчанию Asphyxiation 500), теряет всё. Убери блок `miaDamage` для мягкого режима.
- **KIA-фид**: погибший в рейде выбывает из списка рейдеров с объявлением.
- **Досрочное завершение**: если вышли/погибли все рейдеры — рейд заканчивается, не дожидаясь таймера.
- **Повторяемость рейдов**: контроллер на старте КАЖДОГО рейда чистит лут и скавов прошлого захода
  и заново заселяет локацию через лут-поля (`RaidLootField` для добычи, `RaidSkavField` для скавов).
  Каждый заход стартует с полной, нетронутой локации (а не один свежий рейд за раунд).

## Быстрый запуск (тест без редактора)

Для проверки режима есть тестовый контроллер `RaidControllerTest`: он грузит готовую арену
(`arena_duel_31`) как локацию рейда, ставит короткий таймер (3 мин) и **сам расставляет** на ней
спавны входа, точку экстракта и поля лута/скавов (`autoSetup`). Запуск — пара спавнов через админку:

1. Запусти игру (сервер + клиент), старт раунда, права админа.
2. Открой меню спавна сущностей (F5 / «Spawn entities») и заспавни рядом с собой:
   - `RaidControllerTest` — при спавне грузит карту рейда и авто-настраивает её;
   - `RaidEntryButton` — кнопка входа;
   - `RaidShopTerminal` — магазин (потратить заработанные ТК); по желанию `RaidReturnMarker`.
3. Встань на тот же грид, что и кнопка, и активируй её → телепорт в рейд, пойдёт таймер 3 мин,
   локация заселится лутом и скавами.
4. Убивай скавов, собирай лут, встань на точку экстракта ~10 сек → возврат на хаб с наградой ТК.
5. Не успеешь за 3 минуты — MIA (смерть, потеря всего). Вышли/погибли все — рейд закончится раньше.

> ⚠️ Кнопка входа утаскивает в рейд ВСЕХ мобов со своего грида. Для чистого теста жми её там, где
> рядом нет посторонних (или тестируй в одиночном раунде). `RaidControllerTest` — только для отладки,
> в реальный режим ставь обычный `RaidController` со своей картой.

## Полноценный запуск: карта делается в редакторе

Подтверждено: тайлы карт SS14 — это base64-кодированные бинарные чанки (`MapGrid.chunks[].tiles`),
руками валидную карту не написать. Поэтому `raid_location.yml` и расстановка маркеров — работа в
ин-гейм редакторе (как все наши арены). Без неё режим не запускается (контроллер пишет `Warning`).

Маппинг (минимум на карте рейда): спавны входа, точки экстракта, один `RaidLootField` и один
`RaidSkavField`. Контроллер сам заселит локацию на каждом заходе.
