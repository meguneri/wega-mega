==========================================================
  wega-mega — как маппить (без GitHub и возни с гитом)
==========================================================

ЧТО НУЖНО ОДИН РАЗ
------------------
Установить .NET SDK 10:
  https://dotnet.microsoft.com/download/dotnet/10.0
(нужен именно SDK, не Runtime).

ЗАПУСК
------
  Windows : дважды кликни  run_mapping.bat
  macOS   : в терминале:   ./run_mapping.sh
  Linux   : в терминале:   ./run_mapping.sh

Первый запуск собирает проект — это несколько минут, потом быстро.
Скрипт сам поднимет локальный сервер и откроет игру.

ВОЙТИ В МАППИНГ
---------------
1. В клиенте в поле адреса введи:  localhost   (или 127.0.0.1)
   На локальном сервере ты СРАЗУ полный админ — ничего настраивать не надо.
2. Открой консоль клавишей  ~  (тильда, слева от 1).
3. Команды:

   Создать НОВУЮ пустую карту (id — любое свободное число):
       mapping 50

   Открыть существующую карту для правки:
       mapping 50 Maps/_Wega/Arena/DMarena2urban.yml

4. Маппишь обычными инструментами (спавн, тайлы, декали).

СОХРАНЕНИЕ
----------
В консоли:
    savemap 50 Maps/_Wega/Arena/моя_карта.yml

Файл появится в:
    Resources/Maps/_Wega/Arena/моя_карта.yml
Это обычный текстовый .yml — его и отправляешь обратно автору проекта.

ЧАСТЫЕ ВОПРОСЫ
--------------
* "command not found" / нет админки — убедись, что подключился именно к
  localhost, а не к внешнему адресу.
* Долгая первая сборка — это нормально, .NET компилит весь проект один раз.
* Ничего из GitHub качать не нужно: весь код уже в этой папке.
